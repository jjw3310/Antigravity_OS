# Contributing to Antigravity OS

We are thrilled that you want to contribute! Antigravity OS is a community-driven effort to build the most disciplined AI engineering guidelines.

## 🤝 How to Contribute

### 1. Adding a New Skill Module
Skill modules are stored in `template/skills/[skill-name]/SKILL.md`.
- See our [Skill Creation Guide](./docs/SKILL_CREATION_GUIDE.md) for a detailed walkthrough.
- Each module must follow the YAML frontmatter format.
- Focus on specific, high-value engineering domains (e.g., `react-performance`, `solidity-security`).
- Include a **Surgical Checklist** for the AI agent to verify its own work.

### 2. Improving the Dashboard
The dashboard is a Vanilla HTML/CSS interface.
- Keep the "Premium Aesthetics" (Glassmorphism, dark mode).
- Ensure all CSS is reusable and scalable.

### 3. Reporting Issues & Labeling
We use labels to categorize contributions:
- `good first issue`: Perfect for beginners.
- `help wanted`: Complex tasks needing experienced engineers.
- `new-skill`: Proposal for a new domain expertise.

## 🛡️ Review Process
Every Pull Request will be reviewed against our **Core Principles**:
1. **Surgicality**: Does this change minimize churn?
2. **Aesthetics**: Does it maintain the premium vibe?
3. **Clarity**: Is the instruction unambiguous for an AI agent?

## 🏆 Hall of Fame
All contributors will be featured in the README. We value your time and expertise!
