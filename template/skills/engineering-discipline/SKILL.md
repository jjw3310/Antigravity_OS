# Engineering Discipline Skill

## 📋 Methodology
1. **Identify**: Clearly state the problem and the constraints. **If ambiguous, ask for clarification immediately.**
2. **Scan**: List all relevant files and their current state.
3. **Draft**: Write out the intended logic in pseudo-code or high-level steps.
4. **Communicate**: Surface potential side effects to the user.
5. **Execute**: Implement surgically.

## 🚫 Anti-Patterns to Avoid
- **Blind Copy-Paste**: Do not copy code without understanding its context.
- **Speculative Abstraction**: Do not build frameworks for single-use logic.
- **Silent Failure**: Never leave an empty catch block.
- **Code Deletion**: Never accidentally delete unrelated comments or imports.

## ✅ Completion Checklist
- [ ] Are all variables named descriptively?
- [ ] Did I handle the "sad path" (errors/edge cases)?
- [ ] Is the code more readable than before?
- [ ] Does it perfectly match the project's existing style?
